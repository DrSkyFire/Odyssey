 1. 硬件连接
=============================================================================

【必需连接】
1. 电源:
   - 开发板主电源（通常5V/12V，根据板卡规格）
   - 确保电源稳定，纹波<50mV

2. HDMI显示:
   - HDMI输出 → 显示器/电视
   - 分辨率: 1080p@60Hz
   - 确保显示器支持该分辨率

3. ADC信号输入:
   - 通道1 (CH1): 连接到adc_ch1_data引脚
   - 通道2 (CH2): 连接到adc_ch2_data引脚
   - 输入范围: 0-3.3V (10位ADC, MS9280)
   - 最大采样率: 1MSPS

【可选连接】
4. UART调试:
   - 波特率: 115200
   - 数据位: 8
   - 停止位: 1
   - 校验: 无
   - TX引脚: R9 (LVCMOS33)
   - 用途: 实时监控系统状态

5. 按键:
   - 8个用户按键（user_button[7:0]）
   - 功能详见"按键功能一览"

【信号源建议】
- 频率范围: 1Hz - 500kHz
- 幅度: 0.5V - 3.0Vpp
- 偏置: 1.65V (ADC中点)
- 波形类型: 正弦波、方波、三角波、脉冲

=============================================================================
 2. 上电启动流程
=============================================================================

【自动启动序列】

T = 0s      上电
            ↓
            - FPGA配置加载
            - PLL时钟初始化
            - MS7210 HDMI芯片初始化
            ↓
T = 0.5s    【自动启动】
            ↓
            - run_flag自动置位
            - ADC开始采集
            - 时域波形显示（默认模式）
            ↓
T = 1s      系统稳定运行
            ↓
            用户可操作

【启动完成标志】

1. HDMI屏幕显示:
   ✅ 顶部标题栏: 通道指示（CH1绿色/CH2红色）
   ✅ 中部波形区: 绿色时域波形（如果有信号输入）
   ✅ 底部参数区: 参数文字显示区

2. UART输出（每0.5秒）:
   ```
   HDMI: 1 1 1 1              ← 所有系统正常
   Mode:T Trig:A Lv:512       ← 时域模式，Auto触发
   ADC:1 FIFO:0026 FFT:1 0000 ← ADC工作，FIFO递增
   ```

3. LED指示:
   ✅ LED[0]: 自动点亮（run_flag=1）
   ✅ 其他LED: 根据测试结果显示

【按键操作提示】
• 所有按键都有消抖处理（约20ms）
• 按键为单脉冲触发，无需长按
• UART会实时反馈按键操作结果
• Button[3]和[6]的区别:
  - [3]影响触发源（哪个通道用来判断触发条件）
  - [6]影响显示（屏幕显示哪个通道的数据）
  - 两者独立工作，可任意组合

 5. UART调试输出
=============================================================================

【连接设置】
- 波特率: 115200 bps
- 数据位: 8
- 停止位: 1
- 校验: 无
- 流控: 无

【输出格式】（每0.5秒更新）

时域模式 (Mode:T):
┌─────────────────────────────────────┐
│ HDMI: 1 1 1 1                       │ ← 系统状态
│ Mode:T Trig:A Lv:512                │ ← 工作模式+触发信息
│ ADC:1 FIFO:0026 FFT:1 0000          │ ← 调试信息
└─────────────────────────────────────┘

频域模式 (Mode:F):
┌─────────────────────────────────────┐
│ HDMI: 1 1 1 1                       │
│ Mode:F                              │ ← 无触发信息
│ ADC:1 FIFO:0026 FFT:1 0234          │ ← FFT计数递增
└─────────────────────────────────────┘

测量模式 (Mode:M):
┌─────────────────────────────────────┐
│ HDMI: 1 1 1 1                       │
│ Mode:M                              │
│ ADC:1 FIFO:0026 FFT:1 0000          │
└─────────────────────────────────────┘

【字段说明】

第1行 - HDMI系统状态:
  HDMI: P1 P2 CLK INIT
        ↑  ↑  ↑   ↑
        │  │  │   └─ MS7210初始化完成 (0/1)
        │  │  └───── HDMI时钟活动 (0/1)
        │  └──────── PLL2锁定 (0/1)
        └─────────── PLL1锁定 (0/1)

  正常值: "1 1 1 1" (全部系统就绪)

第2行 - 工作模式:
  Mode:X Trig:Y Lv:ZZZ
       ↑      ↑     ↑
       │      │     └─ 触发电平 (0-1023)
       │      └─────── 触发模式 (A=Auto, N=Normal)
       └────────────── 工作模式 (T/F/M)

第3行 - 调试信息:
  ADC:V FIFO:AAAA FFT:D BBBB
      ↑      ↑       ↑    ↑
      │      │       │    └─ FFT输出计数 (BCD码)
      │      │       └────── FFT完成标志 (0/1)
      │      └──────────────  FIFO写入计数 (BCD码)
      └─────────────────────  ADC数据有效 (0/1)

【故障诊断】

✅ 正常输出:
   HDMI: 1 1 1 1
   Mode:T Trig:A Lv:512
   ADC:1 FIFO:0026 FFT:1 0000
   
   → 所有系统正常，ADC采集中，FIFO递增

❌ PLL未锁定:
   HDMI: 0 1 1 1  或  HDMI: 1 0 1 1
   
   → 时钟系统异常，检查晶振

❌ ADC无数据:
   ADC:0 FIFO:0000 FFT:1 0000
   
   → ADC采集失败，检查信号输入

❌ FIFO不递增:
   FIFO:0026 ... (数值不变)
   
   → 数据流停止，可能run_flag=0或触发未成功

❌ FFT不工作:
   Mode:F
   FFT:0 0000  (计数始终为0)
   
   → FFT模块未运行，检查run_flag或FIFO

 6. 常见使用场景
=============================================================================

【场景0: 理解双通道工作机制】

本系统采用**双通道同步采集**架构，两个通道始终同时工作：

┌─────────────────────────────────────────────────────────────┐
│ 硬件层: 双ADC同步采集                                       │
├─────────────────────────────────────────────────────────────┤
│ CH1(绿) ──┬─> ADC1 @ 1MHz ──> FIFO1 (2048深度)             │
│           │                                                  │
│ CH2(红) ──┼─> ADC2 @ 1MHz ──> FIFO2 (2048深度)             │
│           │                                                  │
│         同步时钟                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 按键[3]: 触发源选择 (仅影响时域模式)                        │
├─────────────────────────────────────────────────────────────┤
│ channel_sel = 0 (默认):                                     │
│   • 使用CH1信号检测触发条件                                 │
│   • 当CH1信号上升沿穿越trigger_level时触发                  │
│   • 两个通道同步捕获1024点数据                              │
│                                                              │
│ channel_sel = 1:                                             │
│   • 使用CH2信号检测触发条件                                 │
│   • 当CH2信号上升沿穿越trigger_level时触发                  │
│   • 两个通道同步捕获1024点数据                              │
│                                                              │
│ 应用场景:                                                    │
│ ✓ 观察CH1和CH2的时序关系（相位、延迟）                       │
│ ✓ 用稳定的信号作为触发源，观察另一个不稳定信号              │
│ ✓ 例: CH1=时钟信号(触发源), CH2=数据信号(被观察)           │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│ 按键[6]: 显示通道选择 (所有模式)                            │
├─────────────────────────────────────────────────────────────┤
│ display_channel = 0 (默认):                                 │
│   • HDMI顶部: [CH1 绿色] [CH2 灰色]                         │
│   • 时域模式: 显示CH1绿色波形                               │
│   • 频域模式: 显示CH1频谱柱状图                             │
│   • 测量模式: 显示CH1参数（Freq/Amp等）                     │
│                                                              │
│ display_channel = 1:                                         │
│   • HDMI顶部: [CH1 灰色] [CH2 红色]                         │
│   • 时域模式: 显示CH2红色波形                               │
│   • 频域模式: 显示CH2频谱柱状图                             │
│   • 测量模式: 显示CH2参数（Freq/Amp等）                     │
└─────────────────────────────────────────────────────────────┘

【关键区别】
┌──────────────┬───────────────┬───────────────────────┐
│ 按键         │ 影响范围      │ 作用                   │
├──────────────┼───────────────┼───────────────────────┤
│ Button[3]    │ 仅时域触发    │ 选择触发判断依据哪路   │
│ CH_SEL       │               │ 数据采集仍然是双通道   │
├──────────────┼───────────────┼───────────────────────┤
│ Button[6]    │ 所有模式      │ 选择HDMI显示哪路数据   │
│ DISP_CH      │               │ 另一路仍在后台处理     │
└──────────────┴───────────────┴───────────────────────┘

【典型示例】
设置: channel_sel=0 (CH1触发), display_channel=1 (显示CH2)
结果: 
  - 系统监视CH1波形，当CH1穿越trigger_level时触发
  - 触发瞬间，两个通道同时捕获数据
  - 屏幕显示CH2的红色波形（同步于CH1的触发点）
  
应用: 观察CH1时钟信号触发时，CH2数据线的状态

---

【场景1: 首次上电，验证ADC工作】

目标: 确认ADC能否正常采集信号

步骤:
1. 连接1kHz正弦波到CH1 (1Vpp, 1.65V偏置)
2. 上电，等待0.5秒自动启动
3. 观察HDMI:
   ✓ 应该看到绿色波形上下跳动
   ✓ 波形应该相对稳定（触发作用）
4. 观察UART:
   ```
   HDMI: 1 1 1 1         ← 系统正常
   Mode:T Trig:A Lv:512  ← 时域模式
   ADC:1 FIFO:0026 ...   ← ADC工作，FIFO递增
   ```

判断:
✅ 波形跳动 + FIFO递增 → ADC正常
❌ 波形静止 + ADC:0 → ADC故障

---

【场景2: 调整触发电平，稳定波形】

目标: 找到最佳触发点，使波形最稳定

步骤:
1. 确保在时域模式（Mode:T）
2. 观察波形:
   - 如果波形左右"漂移" → 需要调节触发电平
3. 调节方法:
   - 按Button[1]增加电平 → Lv:522, 532...
   - 按Button[2]减少电平 → Lv:502, 492...
4. 找到最佳点:
   - 波形完全静止，不再左右移动
   - 通常在信号上升沿中点

技巧:
• 正弦波: 触发电平设在幅度的50%处
• 方波: 触发电平设在阈值电压（如512）
• 脉冲: 触发电平设在脉冲前沿中间

【场景3: 切换到频域，分析频谱】

【场景5: 双通道时序关系分析】

目标: 测量两路信号的相位差或时间延迟

步骤:
1. 连接两路信号到CH1和CH2（例如同一信号源的两路输出）

   - 按Button[3]选择用哪路做触发（通常选择幅度较大的）
   - UART会反馈channel_sel状态（虽然当前未显示）

   - 按Button[6]在CH1和CH2之间切换

   - 计算屏幕上两波形的水平偏移像素数

应用:
• 测量滤波器的相移
• 验证信号传输延迟
• 检查差分信号对的对称性
• 调试时序关系（如时钟与数据）

• 使用CH1作为触发源（稳定的参考信号）
• 显示CH2观察待测信号相对于参考的时序
• Normal触发模式可获得更精确的捕获

【场景8: 测量模式的相位差显示】

注意:
• 两路信号频率必须相同才有意义
• 频率偏差会导致相位差不断变化
• 适合稳定的周期信号

 7. 故障排查
=============================================================================

【问题1: 上电后HDMI无显示】

可能原因:
1. HDMI线缆松动
   → 重新插拔HDMI线

2. 显示器不支持1080p@60Hz
   → 更换显示器或修改分辨率（需重新编译）

3. PLL2未锁定
   → 检查UART: HDMI: 1 0 1 1
   → 检查27MHz时钟源

解决方法:
- 检查sys_clk_27m引脚波形
- 重启FPGA

---

【问题2: UART无输出】

检查项:
1. 串口工具设置:
   ✓ 波特率115200
   ✓ 8N1格式
   ✓ 正确的COM口

2. TX引脚连接:
   ✓ FPGA R9引脚 → USB转串口RX
   ✓ 共地连接

3. 运行状态:
   → run_flag必须为1才有UART输出

---

【问题3: 时域波形滚动，不稳定】

原因: 触发电平设置不当

解决:
1. 检查UART显示的Lv值
2. 调节触发电平:
   - 信号幅度大 → 调高电平
   - 信号幅度小 → 调低电平
3. 切换到Auto模式（Trig:A）
4. 如果信号本身不稳定，任何触发都无法完全稳定

---

【问题4: FFT无输出（计数不增）】

检查项:
1. 工作模式:
   → 必须在频域模式（Mode:F）

2. 运行标志:
   → run_flag=1 (按Button[1]启动)

3. FIFO数据:
   → UART显示FIFO计数应该递增
   → 如果FIFO为0，说明ADC没有数据

4. FFT使能:
   → 检查fft_start信号
   → fft_start = run_flag && (work_mode == 2'd1)

---

【问题5: 波形幅度异常】

现象: 波形很小或很大

原因:
1. 输入信号超出0-3.3V范围
   → 调整信号源偏置和幅度

2. ADC参考电压异常
   → 检查ADC供电

3. 时域显示缩放:
   → 当前缩放固定，无法调节
   → 未来版本可添加增益控制

---

【问题6: 参数测量不准确】

原因:
1. 信号频率过高
   → 采样率1MHz，最高测500kHz
   → 降低信号频率

2. 信号不稳定
   → FFT需要稳定周期信号
   → 检查信号源质量

3. FFT分辨率限制
   → 频率分辨率约976Hz
   → 测量1kHz信号时误差较大

 9. 微弱信号检测功能 (Lock-in Amplifier)
=============================================================================

【功能概述】
本系统集成了数字锁相放大器（Lock-in Amplifier），用于从强噪声背景中
提取微弱的周期性信号。该功能基于相干检测原理，通过混频和低通滤波
实现高灵敏度的信号检测。

【检测原理】
1. 正交解调: 信号与参考信号(sin/cos)相乘
2. 低通滤波: 移除高频噪声，保留目标频率分量
3. I/Q提取: 同相(I)和正交(Q)分量独立输出
4. 幅度/相位: 从I/Q计算信号幅度和相位

               输入信号
                  ↓
            [数字增益控制]
                  ↓
        ┌─────────┴─────────┐
        ↓                    ↓
    [混频×cos(ωt)]      [混频×sin(ωt)]
        ↓                    ↓
    [低通滤波]           [低通滤波]
        ↓                    ↓
     I通道                Q通道
        └─────────┬─────────┘
                  ↓
         [幅度/相位计算]

【工作模式】
模式0: 内部DDS - 可编程频率合成器生成参考信号
模式1: CH2作参考 - 使用通道2输入作为参考信号
模式2: 外部参考 - 外部输入参考信号（预留）
模式3: 自动搜索 - 自动频率跟踪（未实现）

【配置参数】(在代码中设置)
1. 参考频率 (ref_frequency)
   - 范围: 100 Hz ~ 17.5 MHz
   - 默认: 1000 Hz (1kHz)
   - 步进: 100 Hz

2. 数字增益 (digital_gain)
   - 范围: 0 ~ 15
   - 对应: 1x ~ 32768x
   - 默认: 4 (16x增益)
   - 建议: 开启自动增益控制(AGC)

3. 滤波器时间常数 (lpf_time_constant)
   - 范围: 6 ~ 12
   - 对应: 64 ~ 4096点移动平均
   - 默认: 8 (256点)
   - 说明: 越大越慢但越精确

4. 自动增益控制 (auto_gain_enable)
   - 默认: 开启
   - 功能: 自动调整增益到最佳范围
   - 调整周期: 每256个样本

【输出信号】
1. I/Q分量 (ch1_lia_i, ch1_lia_q)
   - 位宽: 24位有符号数
   - 用途: 精密相位分析

2. 幅度 (ch1_lia_magnitude)
   - 计算: mag = sqrt(I² + Q²)
   - 单位: 相对值
   - 用途: 信号强度检测

3. 相位 (ch1_lia_phase)
   - 范围: 0 ~ 65535 (对应0°~360°)
   - 精度: ±1°
   - 用途: 相位差测量

4. 锁定状态 (ch1_lia_locked)
   - 判据: 幅度波动 < 12.5%
   - 用途: 信号稳定性指示

5. SNR估计 (weak_sig_snr)
   - 单位: dB
   - 更新周期: 1000个样本
   - 用途: 信号质量评估

【使用步骤】
1. 硬件连接:
   - CH1: 待检测微弱信号
   - CH2: 参考信号(如使用模式1)

2. 代码配置:
   修改 signal_analyzer_top.v 中的初始值:
   
   weak_sig_enable = 1'b1;           // 使能检测器
   weak_sig_ref_mode = 2'b00;        // 选择参考模式
   weak_sig_ref_freq = 32'd1000;     // 设置1kHz参考频率
   weak_sig_auto_gain = 1'b1;        // 开启AGC
   weak_sig_lpf_tc = 4'd8;           // 256点滤波

3. 重新综合并下载FPGA比特流

4. 观察输出:
   - 查看 ch1_lia_magnitude (幅度)
   - 检查 ch1_lia_locked (锁定状态)
   - 读取 weak_sig_snr (信噪比)

【应用场景】
1. 微弱信号检测
   场景: 1mV信号 + 100mV噪声 (SNR = -40dB)
   效果: 可恢复信号幅度和相位
   
2. 相位敏感检测
   场景: 测量两个同频信号的相位差
   方法: phase_diff = ch1_lia_phase - ch2_lia_phase
   
3. 频率选择性测量
   场景: 仅检测特定频率分量
   效果: 抑制所有非目标频率的干扰
   
4. 调制信号解调
   场景: AM/FM调制信号检测
   方法: I/Q分量可重构原始调制信息

【性能指标】
- 最小可检测信号: -60dB (相对满量程)
- 频率精度: ±0.01%
- 相位精度: ±1°
- 动态范围: >80dB (AGC开启)
- 锁定时间: < 10×滤波时间常数
- 等效带宽: Fs / (2^lpf_tc)

【典型配置示例】
1. 高灵敏度模式:
   digital_gain = 12 (4096x)
   lpf_tc = 10 (1024点滤波)
   → 适合: 极微弱信号，可容忍慢速响应

2. 快速响应模式:
   digital_gain = 6 (64x)
   lpf_tc = 6 (64点滤波)
   → 适合: 中等强度信号，需要快速跟踪

3. 平衡模式:
   digital_gain = 8 (256x)
   lpf_tc = 8 (256点滤波)
   → 适合: 大多数应用场景

【注意事项】
1. 参考频率精度
   - 偏差应 < 0.1% 否则无法锁定
   - 建议使用内部DDS或稳定的外部参考

2. 输入范围
   - 确保输入信号在ADC量程内
   - 过大会饱和，过小会淹没在量化噪声中

3. 滤波器延迟
   - 时间常数越大，锁定时间越长
   - 锁定时间 ≈ (2^lpf_tc) / 采样率

4. 资源占用
   - 双通道检测约占用30%逻辑资源
   - 包括2个DDS、4个乘法器、2个CIC滤波器

5. 时钟域
   - 检测器工作在clk_fft域（100MHz）
   - 输入数据从clk_adc域（35MHz）同步

【调试技巧】
1. 验证锁定:
   - locked信号应稳定为高
   - magnitude应稳定不跳变

2. 调整增益:
   - magnitude应在0x010000 ~ 0x700000范围
   - 过小：增加digital_gain
   - 过大：减小digital_gain

3. 优化滤波:
   - 噪声大：增加lpf_tc
   - 响应慢：减小lpf_tc

4. 参考频率扫描:
   - 如不确定信号频率，可小范围扫描ref_frequency
   - 观察magnitude峰值确定准确频率

【相关文档】
- WEAK_SIGNAL_DETECTION.md - 详细技术说明
- lock_in_amplifier.v - 核心算法实现
- weak_signal_detector.v - 顶层控制模块

=============================================================================
 第10章：AI信号识别功能
=============================================================================

【功能概述】
AI信号识别系统利用FPGA的并行计算能力，实现实时波形自动分类。
通过提取8个时域+频域特征，使用决策树算法自动识别以下信号类型：
  • 正弦波 (Sine Wave)
  • 方波 (Square Wave)
  • 三角波 (Triangle Wave)
  • 锯齿波 (Sawtooth Wave)
  • 噪声信号 (Noise)

【核心特点】
1. 实时识别: 延迟<3ms（1024点窗口）
2. 双通道独立: CH1和CH2同时工作
3. 置信度输出: 0-100%可信度评分
4. 硬件加速: 8个特征并行提取
5. 低资源占用: <5K LUT（双通道）

=============================================================================
 10.1 工作原理
=============================================================================

【识别流程】
  采集信号 → 特征提取 → 决策树分类 → 输出结果
     ↓          ↓            ↓           ↓
  1024点    8个特征      阈值判断    波形类型
  @35MHz   并行计算     多级决策    + 置信度

【提取的8个特征】
1. 过零率 (ZCR)         - 信号符号变化频率
2. 峰值因子 (CF)        - Peak / RMS
3. 波形因子 (FF)        - RMS / Mean
4. 平均值 (Mean)        - 直流分量
5. 标准差 (Std)         - 信号波动
6. 总谐波失真 (THD)     - 谐波能量占比
7. 频谱质心 (SC)        - 频率重心
8. 频谱展宽 (SS)        - 频率分散度

【决策树算法】
第1级 - THD粗分类:
  THD < 5%  → 正弦波候选
  THD > 30% → 方波/锯齿波候选
  THD 10-25% → 三角波候选
  THD > 60% → 噪声

第2级 - 峰值因子精细分类:
  CF ≈ 1.0 → 方波
  CF ≈ 1.4 → 正弦波
  CF ≈ 1.7 → 三角/锯齿波

第3级 - 过零率区分:
  ZCR高 → 锯齿波
  ZCR低 → 三角波

 10.2 信号定义与接口
=============================================================================

【输出信号】
┌────────────────────┬──────────┬─────────────────────────┐
│     信号名          │  位宽    │     说明                │
├────────────────────┼──────────┼─────────────────────────┤
│ ch1_waveform_type   │  3 bit   │ CH1波形类型 (0-5)       │
│ ch1_confidence      │  8 bit   │ CH1置信度 (0-100%)      │
│ ch1_ai_valid        │  1 bit   │ CH1结果有效标志         │
├────────────────────┼──────────┼─────────────────────────┤
│ ch2_waveform_type   │  3 bit   │ CH2波形类型             │
│ ch2_confidence      │  8 bit   │ CH2置信度               │
│ ch2_ai_valid        │  1 bit   │ CH2结果有效标志         │
└────────────────────┴──────────┴─────────────────────────┘

【波形类型编码】
  3'd0 = UNKNOWN    (未知信号)
  3'd1 = SINE       (正弦波)
  3'd2 = SQUARE     (方波)
  3'd3 = TRIANGLE   (三角波)
  3'd4 = SAWTOOTH   (锯齿波)
  3'd5 = NOISE      (噪声)

【调试特征输出】（16位）
  ch1_dbg_zcr          - CH1过零率
  ch1_dbg_crest_factor - CH1峰值因子 (Q8.8定点数)
  ch1_dbg_thd          - CH1总谐波失真 (%)
  ch2_dbg_zcr          - CH2过零率
  ch2_dbg_crest_factor - CH2峰值因子
  ch2_dbg_thd          - CH2 THD

 10.3 使用方法
=============================================================================

【基本使用】
1. 默认状态: AI识别功能默认开启 (ai_enable = 1)

2. 读取识别结果:
   - 监听 ch1_ai_valid 信号
   - 当valid=1时，读取waveform_type和confidence
   - 结果每30ms更新一次（约33Hz刷新率）

3. 判断置信度:
   - confidence >= 80: 高可信结果
   - confidence 60-79: 中等可信
   - confidence < 60:  低可信，建议忽略

【Verilog代码示例】
// 识别结果处理
always @(posedge clk) begin
    if (ch1_ai_valid) begin
        case (ch1_waveform_type)
            3'd1: display_text("SINE WAVE");
            3'd2: display_text("SQUARE WAVE");
            3'd3: display_text("TRIANGLE WAVE");
            3'd4: display_text("SAWTOOTH WAVE");
            3'd5: display_text("NOISE");
            default: display_text("UNKNOWN");
        endcase
        
        // 仅当置信度高时采取行动
        if (ch1_confidence >= 8'd80) begin
            case (ch1_waveform_type)
                3'd1: set_trigger_sine();
                3'd2: set_trigger_square();
                // ...
            endcase
        end
    end
end

【按键控制（预留）】
user_button[7] - AI识别开关（当前未启用）
  按下切换: ai_enable <= ~ai_enable

启用方法：在 signal_analyzer_top.v 中修改：
  assign btn_ai_enable = 1'b0;  // 改为实例化消抖模块

=============================================================================
 10.6 故障排查
=============================================================================

【问题1: 始终识别为UNKNOWN】
症状: waveform_type 总是 3'd0
原因: 
  • 信号幅度过小（< 20% FS）
  • FFT数据无效
  • 特征值超出所有阈值范围

解决方案:
  ☑ 检查ADC输入信号幅度（建议0.5-3V）
  ☑ 验证FFT模块是否正常工作
  ☑ 读取 dbg_* 调试信号，分析特征值

【问题2: 置信度始终很低（<50%）】
症状: confidence < 50
原因:
  • 信号质量差（高噪声）
  • 信号频率过高/过低
  • 波形失真严重

解决方案:
  ☑ 提高输入信号幅度
  ☑ 检查信号频率在100Hz-10MHz范围内
  ☑ 降低噪声干扰（屏蔽、滤波）
  ☑ 检查信号源质量

【问题3: 误识别（例如三角波识别成锯齿波）】
症状: 识别结果与实际不符
原因:
  • 阈值不适配当前信号
  • 信号波形不标准
  • 采样率不足（高频信号）

解决方案:
  ☑ 调整 waveform_classifier.v 中的阈值参数
  ☑ 增大分析窗口（1024 → 2048）
  ☑ 提高采样率（如果硬件支持）

【问题4: 方波识别为正弦波】
症状: 理想方波被识别为正弦波
原因: 方波经过低通滤波后THD降低
解决方案:
  ☑ 检查信号路径是否有滤波器
  ☑ 降低SINE_THD_MAX阈值（从5%到3%）
  ☑ 增加方波判定的ZCR权重

【问题5: ch1_ai_valid 始终为0】
症状: 结果从不更新
原因:
  • ai_enable = 0（被禁用）
  • 时钟域问题
  • FFT数据未连接

解决方案:
  ☑ 检查 ai_enable 信号状态
  ☑ 验证 dual_data_valid 信号
  ☑ 检查FFT模块输出连接

 10.9 技术细节
=============================================================================

【定点数格式】
峰值因子和波形因子使用Q8.8格式：
  • 高8位: 整数部分
  • 低8位: 小数部分
  
  示例: 16'h0168 = 1.40625
        16'h0100 = 1.0
        16'h01B0 = 1.6875

【THD计算方法】
  THD = √(H2² + H3² + H4² + H5²) / H1 × 100%
  
  实现简化：THD = (H2 + H3 + H4 + H5) / H1 × 100%
  误差: <5%（可接受）

【特征提取状态机】
  IDLE → COLLECTING → COMPUTE → OUTPUT → IDLE
    ↓       ↓           ↓         ↓
  等待    采集1024点  计算特征   输出结果

【决策树深度】
  当前: 3级决策树
  扩展性: 可增加到5级（需修改分类器）
