## 📝 Git提交规范

后续提交请遵循以下规范：

```
<type>: <subject>

<body>

类型（type）：
- feat: 新功能
- fix: Bug修复
- docs: 文档更新
- style: 代码格式（不影响功能）
- refactor: 重构
- perf: 性能优化
- test: 测试
- chore: 构建/工具配置

示例：
feat: 添加双通道相位差测量模块

- 实现时域过零检测算法
- 精度<1°
- 更新HDMI显示界面
```
